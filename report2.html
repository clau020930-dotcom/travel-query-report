旅游搜索 Query 需求分析报告

数据来源：小红书搜索日志 | 时间范围：2026.06.25–07.01 | 数据类型：搜索 query（问一问曝光） | Session: 20260703_100000_travel

0. 一页纸结论

2,724 条旅游搜索 query 的全局账本：清洗后 2724 条 query 中旅游相关 2684 条（98.5%），非旅游 39 条（1.4%）。问一问整体曝光率 95.5%，点击率 12.6%（323/2563）。以下五个发现按数据置信度从高到低排列。

1. 「XX旅游攻略」是绝对核心搜索模式（874 条，占 32.6%），但问一问点击率仅 12.6%
L2-1 综合旅游攻略以 874 条占据近三分之一搜索量。典型 query 如「杭州旅游攻略」「青岛游玩攻略三天两夜」。然而问一问在这个场景的 CTR 仅 12.6%（106/844），说明用户搜「攻略」时更倾向于浏览 UGC 笔记而非阅读 AI 摘要——这与购物场景中品牌探索型用户的行为模式一致。但这不意味着问一问无价值：攻略 query 的 CTR 虽低于平均，绝对点击量（106 次）仍是所有 L2 中最高的。
2. 签证/实用信息类 query 的问一问 CTR 显著高于攻略类（21.7%/20.0%/17.7%），是 AI 效果最好的场景
L2-10 签证与证件（CTR 21.7%）、L2-6 旅途餐饮（CTR 20.0%）、L2-9 自驾与租车（CTR 17.7%）——这些场景的共性是用户需要确定性答案（签证政策、能否携带物品、自驾规则），而非视觉化的灵感浏览。相比之下，L2-11 旅行装备与穿搭（CTR 5.0%）、L2-12 旅游文案与素材（CTR 7.4%）、L2-5 酒店与民宿（CTR 7.5%）的 CTR 最低，说明装备穿搭和文案素材类搜索是视觉驱动的，AI 摘要无法替代图文笔记。
3. 攻略→景点→酒店构成旅游决策链的三大支柱（合计 60.4%），但当前是独立搜索而非连贯流程
L2-1 综合攻略（32.6%）+ L2-4 景点推荐（14.9%）+ L2-5 酒店民宿（12.9%）合计占 60.4%。用户的真实需求是一条决策链：选目的地 → 找景点 → 订住宿，但当前每条搜索被独立处理，丢失了链路上下文。这与购物决策中「品类探索 → 品牌锁定 → 对比决策」的模式同构。
4. 搜索词以短词为主（P50=8 字符），用户意图高度压缩，AI 理解和展开是关键
90% 的搜索词在 12 字符以内，大量 query 是「地名+攻略」「地名+好玩的」的极简模式。用户用最少的字表达模糊的意图，期望搜索系统展开为丰富的内容。这意味着 AI 理解短 query 背后的隐含需求（时间、预算、人群偏好）是提升体验的关键杠杆。
5. 87.4% 的问一问曝光未被点击——曝光率不是瓶颈，内容匹配度才是
问一问整体曝光率高达 95.5%（2563/2684），但点击率仅 12.6%。问题不在于展示不够，而在于展示了但用户不点。可能的原因：(1) 攻略类 query 用户更信任 UGC 笔记；(2) 问一问摘要缺乏图文吸引力；(3) 部分场景的 AI 回答精准度不够。CTR 的 L2 间分化（5.0%~21.7%）说明这是一个内容匹配度问题，而非系统性的产品问题。

问一问关键指标口径

本报告中「问一问曝光率」指该 query 触发了问一问展示的比例，「问一问 CTR」指在曝光的 query 中用户点击了问一问的比例。需注意：CTR 衡量的是 AI 摘要对用户的吸引力，不等于「AI 回答是否满足了用户需求」。用户可能点击后发现答案不满意，也可能不点击但通过扫读摘要获取了关键信息（「免点击满足」）。本数据无法区分这两种情况。

局限性

  1. LLM 自动标注非人工标注，抽样 20 条复核准确率约 95%（19/20），系统性偏差无法排除。特别是 L2-2 路线规划与 L2-9 自驾、L2-5 酒店与 L2-13 实用信息的边界存在灰色地带。
  2. 单一入口类型：99%+ 的 query 来自 dqa_onebox_card 入口，样本仅覆盖问一问触达的搜索场景，不代表小红书全站旅游搜索需求。
  3. 无 session 关联：每条 query 独立记录，无法重建用户的搜索序列和决策路径。报告中关于「决策链」的推断基于 query 模式归纳，非直接观测。
  4. 7 天窗口的季节性偏差：6月底–7月初正值暑期出行规划高峰，攻略类搜索可能被高估。冬季数据中温泉/滑雪/春节出行可能改变 L2 分布。
  5. 用户画像为平台级标签：性别、年龄段、城市等级为平台推断值，存在一定误差。且同一用户仅出现一条 query，无法分析用户级别的搜索行为模式。

用户意图解读

薄弱点一：「攻略是核心需求」可能是分类粒度的假象

部分成立。需要区分「规划型」（如「青岛游玩攻略三天两夜」含时间约束）和「入口型」（如「杭州旅游攻略」纯泛搜索）。粗略估计后者占 L2-1 的 60% 以上。

不建议投入的方向

旅行装备/穿搭场景的 AI 摘要优化（L2-11，CTR 5.0%）
用户搜索穿搭和装备时要的是「看图找灵感」，这是视觉驱动的行为。AI 文字摘要在这个场景中几乎无价值——用户需要的是好看的穿搭笔记图片，而非「海边建议穿白色连衣裙」的文字描述。投入 AI 优化的 ROI 极低。
旅游文案/素材场景的 AI 生成（L2-12，CTR 7.4%）
用户搜索「海边文案」「旅游背景音乐」时，需求本质是内容创作素材，不是旅游信息获取。这个场景应归属于内容创作工具链，而非旅游 AI 助手。且搜索量仅 74 条（2.8%），不值得专项投入。
在无 session 数据的情况下做多轮对话引导
报告中关于「决策链」的推断尚缺乏 session 级证据。在没有验证用户确实按「攻略→景点→酒店」路径搜索之前，贸然设计跨搜索的上下文继承功能，风险大于收益。建议先获取 session 数据验证决策链假设。

底线判断

2684 条旅游搜索 query 中，问一问的首要瓶颈不是曝光率（95.5%），而是内容匹配度（CTR 12.6%）。信息确认类场景（签证、交通规则、实用信息）已经验证了 AI 摘要的价值（CTR 17~22%），攻略和推荐类场景的低 CTR 说明问一问需要从「文字摘要」升级为「结构化方案+视觉内容」。正确的路径是：先在 CTR 高的场景深耕（保证质量，建立信任），同时用结构化升级提升攻略场景的 CTR。不建议在 CTR 极低的视觉消费场景（穿搭、文案）投入 AI 优化。
